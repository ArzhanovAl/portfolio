# Portfolio site

Static portfolio website for GitHub Pages.

## Files

- `index.html` - main portfolio page
- `styles.css` - website styles
- `assets/` - optimized images used by the site
- `hh-profile.md` - text blocks for HH.ru and cover letters

## Portfolio logic

The page is structured as a hiring presentation:

1. Clear role: AI-first designer and marketplace content specialist.
2. Ozon/card examples as commercial proof.
3. Rich content as brand/storytelling proof.
4. AI scene as generative visual proof.
5. Process and tool stack.
6. Personal block and contacts.

The site is intentionally strict and neutral: the examples are visually bright, so the interface should not compete with them.

## How to publish on GitHub Pages

1. Create a new public repository on GitHub.
2. Upload all files from this folder to the repository root.
3. Open repository settings.
4. Go to **Pages**.
5. In **Build and deployment**, choose **Deploy from a branch**.
6. Select branch `main` and folder `/root`.
7. Save.

The site will become available at:

```text
https://YOUR_USERNAME.github.io/REPOSITORY_NAME/
```

If the repository is named `YOUR_USERNAME.github.io`, the site will be available at:

```text
https://YOUR_USERNAME.github.io/
```
